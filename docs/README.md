# Docs

## Activo

- `manuales/`: manuales operativos y sus fuentes de generación.

## Archivo

- `archive/`: material legacy o histórico que ya no forma parte del runtime ni del flujo principal de desarrollo.

## Criterio

- El código y assets necesarios para la app activa quedan en `client/`, `server/`, `shared/` y configuración raíz.
- La documentación histórica o de trabajo exploratorio se mueve a `docs/archive/` en lugar de quedar mezclada en la raíz.
